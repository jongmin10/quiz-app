`src/data/questions.js`에 새 문제를 추가해줘. 아래 순서를 따라줘.

## 1. 현재 구조 파악

`src/data/questions.js`를 읽고 사용 가능한 카테고리와 난이도를 확인해줘.

- 카테고리: 한국사 / 과학 / 지리 / 예술과 문화
- 난이도: 하 / 중 / 상

## 2. 문제 정보 수집

사용자가 아래 정보를 $ARGUMENTS로 제공하지 않았다면 **한 번에 모두 물어봐줘**.

| 항목 | 설명 |
|------|------|
| 카테고리 | 위 4개 중 하나 |
| 난이도 | 하·중·상 중 하나 |
| 문제 | 질문 문장 |
| 보기 4개 | 선택지 (정답 포함) |
| 정답 번호 | 1·2·3·4 중 하나 |
| 해설 | 정답 근거 설명 |

$ARGUMENTS: $ARGUMENTS

## 3. CLAUDE.md 가이드라인 검증

추가 전에 아래 4가지를 반드시 확인하고, 문제가 있으면 **수정안을 제시한 뒤 사용자 확인**을 받아줘.

1. **정답 유일성** — 다른 보기가 정답이 될 수 있는가? 모호하면 조건을 문제에 명시해.
2. **최상급 기준** — "가장 큰", "최초의" 등 표현에 측정 기준(면적·해발고도·연도 등)이 있는가?
3. **시간·범위 명확성** — 변할 수 있는 정보(순위·기록)엔 시점을, 지리·분류엔 범위를 명시해.
4. **교차 검증** — 논란 여지가 있는 내용은 주류 학설 기준임을 해설에 표기해.

검증 결과를 아래 형식으로 출력해줘.

- ✅ 통과 항목
- ⚠️ 수정 권장 항목 (수정안 포함)
- ❌ 오류 항목 (반드시 수정 후 진행)

## 4. questions.js에 삽입

검증을 통과하면 해당 카테고리·난이도 배열의 **마지막**에 아래 형식으로 추가해줘.

```js
{
  question: '문제 문장',
  options: ['보기1', '보기2', '보기3', '보기4'],
  answer: 0,   // 0-based index
  explanation: '해설 문장',
},
```

## 5. 완료 보고

추가된 문제의 최종 형태를 코드 블록으로 보여주고, 몇 번째 문제로 삽입됐는지 알려줘.
